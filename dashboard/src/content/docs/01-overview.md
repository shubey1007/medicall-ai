# MediCall AI — Overview

MediCall AI is an AI-powered voice receptionist built for medical clinics. It answers inbound phone calls around the clock, triages patient symptoms, books appointments, answers medication questions, and escalates genuine emergencies — all without a human picking up the phone. The system has two voice entry points: a production-grade **Twilio** inbound phone path (patients call a real phone number) and a **Vapi** browser-demo path that lets judges, stakeholders, or developers try the assistant directly from a web browser via WebRTC, with auto-detection of Hindi and English. Both paths feed the same multi-agent backend, and a real-time **React + Socket.IO** staff dashboard shows live call transcripts, agent handoffs, and urgency flags so clinic staff stay in the loop without being on the phone.

The platform is not a chatbot bolted onto a phone system. It uses the **OpenAI Realtime API** (`gpt-4o-realtime-preview`), which combines speech-to-text, language model reasoning, and text-to-speech into a single low-latency WebSocket connection — the same connection that carries the patient's voice carries the AI's reply, eliminating the extra round-trip that traditional STT → LLM → TTS pipelines incur. Patient history is stored in **Qdrant**, a cloud-hosted vector database, so the assistant recognises returning callers and avoids asking them to repeat themselves. Appointment and patient records live in **PostgreSQL**, and every call gets an automated post-call summary generated by `gpt-4o-mini` for the clinic's records.

## The Problem

A busy outpatient clinic can receive hundreds of calls per day. The majority are routine: "Can I book an appointment?", "What is my prescription dosage?", "I have a fever — should I come in?" Each call takes a human receptionist two to five minutes, and during peak hours the hold queue grows until patients hang up and simply don't come back. English-only front-desk staff are a bottleneck for multilingual patient populations, and the cognitive load of simultaneously managing in-person check-ins, walk-ins, and a ringing phone leads to errors and staff burnout. Voice AI is the right solution precisely because patients already expect to use the phone — no app to download, no portal to log in to, no behaviour change required on the patient's side.

## What It Does

- **Takes phone calls 24/7** — patients call a Twilio number and speak to an AI receptionist
- **Triages symptoms** — TriageAgent assesses urgency (low / medium / high / critical)
- **Books appointments** — SchedulingAgent can check availability and create bookings
- **Answers medication questions** — MedicationAgent uses a curated database and RAG-backed medical knowledge from Qdrant
- **Detects emergencies** — EmergencyAgent escalates to 911 and notifies the on-call clinician via SMS
- **Remembers patients across calls** — Qdrant `patient_memory` collection stores per-patient episodic facts extracted after every call
- **Multilingual support** — the Vapi path supports Hindi and English auto-detection so patients can speak naturally
- **Browser demo** — judges and stakeholders can try the AI from a web browser via the `@vapi-ai/web` WebRTC SDK, no phone needed
- **Real-time dashboard** — staff see live calls, rolling transcripts, urgency levels, and agent handoffs as they happen

## One-Paragraph Pitch

Clinics are drowning in routine phone calls. A human receptionist answering "Can I book Tuesday at 3?" for the hundredth time is an expensive, error-prone bottleneck — and when that receptionist is on the phone, real emergencies can slip through. MediCall AI replaces that first line of phone contact with an always-on, always-polite AI that triages symptoms, books appointments, answers medication questions, and — critically — recognises a true emergency and immediately notifies the on-call clinician by SMS while staying on the line with the patient. It runs on a real Twilio phone number, persists patient memory across calls, supports Hindi and English, and exposes a real-time staff dashboard so the clinic never loses situational awareness. The one-sentence takeaway: MediCall AI turns the clinic phone from a staff burden into a 24/7 intake system that escalates exactly the right calls to exactly the right people.

## Key Tech

- **FastAPI** — async Python backend and WebSocket server
- **OpenAI Realtime API** (`gpt-4o-realtime-preview`) — single-WebSocket STT + LLM + TTS
- **Twilio Media Streams** — inbound phone calls over WebSocket
- **Vapi** — browser-demo and multilingual voice path via `@vapi-ai/web`
- **Qdrant** — cloud vector database for patient memory, medical knowledge RAG, and doctor directory
- **PostgreSQL** — relational store for patients, calls, appointments, and transcripts
- **React 18 + Vite** — staff-facing real-time dashboard
- **Socket.IO** — pushes live call events from backend to dashboard
